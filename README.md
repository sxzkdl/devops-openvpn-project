# devops-openvpn-project

# Финальная работа: Развертывание и автоматизация безопасной VPN-инфраструктуры

## Общее описание проекта
Данный проект выполнен в рамках финальной работы курса «Старт в DevOps»[cite: 1]. Цель работы — спроектировать и автоматизировать развертывание безопасной сетевой инфраструктуры для ИТ-компании[cite: 1]. 

В ходе проекта была решена задача организации безопасного удаленного доступа сотрудников во внутреннюю сеть через VPN с использованием собственного Удостоверяющего Центра (PKI) для проверки SSL-сертификатов[cite: 1]. Все этапы настройки серверов автоматизированы с помощью Bash-скриптов для обеспечения воспроизводимости инфраструктуры[cite: 1].

---

## Общие требования и инструменты
* **Облачная платформа:** Google Cloud Platform (GCP).
* **Базовая ОС:** Виртуальные машины Compute Engine на базе **Debian 11 (Bullseye)**.
* **Используемое ПО:** `easy-rsa`, `openvpn`, `iptables`.
* **Сетевая защита:** Настройка правил брандмауэра GCP (входящий порт **UDP 1194** для VPN) и локальных фаерволов на серверах.

---

## Архитектура и состав серверов

Проект реализован на базе двух изолированных виртуальных машин в облаке:

| Имя сервера | Роль в сети | Установленные компоненты |
| **ca-server** | Изолированный Удостоверяющий Центр (CA) | `easy-rsa` (инициализация PKI, выпуск корневого сертификата) |
| **vpn-server** | Шлюз OpenVPN | `openvpn` (маршрутизация трафика клиентов, NAT/Маскарадинг) |

## Выполненные задания и автоматизация

### 1) Задание 1: Развертывание Удостоверяющего Центра (PKI)
На первом этапе была развернута изолированная машина `ca-server`[cite: 1]. Процесс инициализации инфраструктуры публичных ключей полностью автоматизирован[cite: 1].

* **Скрипт автоматизации:** `auto-ca.sh` — автоматически обновляет систему, устанавливает `easy-rsa`, конфигурирует файл переменных `vars` и выпускает корневой сертификат `ca.crt` в режиме `nopass`[cite: 1].
* **Артефакты (в папке screenshots/):**
  * `01_auto_ca_code.png` — исходный код bash-скрипта.
  * `02_ca_cert_created.png` — подтверждение успешной генерации корневого сертификата `ca.crt`.

### 2) Задание 2: Создание и настройка VPN-сервера
На втором этапе в той же сети развернута машина `vpn-server`[cite: 1]. На ней настроен демон OpenVPN, принимающий шифрованный трафик от удаленных сотрудников[cite: 1].

* **Скрипт автоматизации:** `setup-vpn.sh` — автоматизирует установку OpenVPN, настраивает сетевой форвардинг пакетов (`sysctl`), конфигурирует правила фаервола и включает NAT (маскарадинг) через `iptables` для выпуска клиентов в интернет[cite: 1].
* **Артефакты (в папке screenshots/):**
  * `03_setup_vpn_code.png` — исходный код скрипта настройки VPN.
  * `04_openvpn_connected.png` — скриншот успешного подключения клиента в программе OpenVPN Connect.
  * `05_yandex_ip_check.png` — скриншот Яндекс.Интернетометра, подтверждающий изменение внешнего IP-адреса клиента на IP-адрес VPN-сервера (регион: Алматы)[cite: 1].
